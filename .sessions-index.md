## How to Use This Index

### Starting a New Project
1. Use the skill: "I want to start a new discovery project"
2. Provide project name, problem statement, and team
3. A new session file is created in `.sessions/`
4. This index is updated with your project

### Resuming an Existing Project
1. Use the skill: "I want to resume discovery work"
2. The skill shows active projects from this index
3. Select the project and stage to resume
4. Session file is loaded with your progress notes

### Monitoring Progress
- Check this index to see all projects at a glance
- Current Stage shows where each project is in the workflow
- Status column indicates: Not Started, In Progress, Ready for Next Stage, Complete

## Session Files Location

Detailed session files for each project live in `.sessions/[project-slug].md`

Example structure:
```
.sessions/
├── mobile-checkout.md          # [Project Name]: Mobile Checkout Discovery
├── subscription-model.md        # [Project Name]: Subscription Model Exploration
└── [project-slug].md           # [Project Name]: [Project Description]
```

Each session file contains:
- Completion status for all 6 stages
- Files created in each stage
- Current work notes and next steps
- Team info and problem statement
- Timestamps for when each stage was completed

## Notes

- Session files are created automatically when you start a project
- Files are updated after each stage completes
- You can pause and resume any project without losing progress
- Timestamps help track when work was done
- Use this index to switch between multiple concurrent projects
